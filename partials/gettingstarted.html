About Potion

Potion is a CSS framework built by me for me. It's a culmination of years of building client projects where I found myself retyping code, looking for snippets I'd used before, and generally just wasting time with initial setup. It's clean, efficient and most importantly... speed optimized. Potion is built it on top of SASS and utilizes AngularJS and it's core for single page app routing, creating custom DOM elements and more.

Click here for details on browser support.

Don't get me wrong, I love some of the bigger frameworks out there, but they can tend to get heavy and bloated, and to be honest... I just wanted something I could call my own. Potion is clean, minimal, mobile first and speed optimized. It only includes the things I find myself using most in my day to day projects... grids, general layout, styling and some very basic components like modals, buttons, etc.

Feel free to use it if you find value, or fork it off of Github and give me a pull request if you have ways to improve! Please just keep in mind that potion is still in it's infancy. I have many future enhancements planned, so I highly suggest decoupling your own styles from the core of Potion so that with future upgrades, you have less headache.

Quick Install with Bower

Potion is available for install via Bower Package Manager. Assuming you already have Node, Bower and Gulp installed, all you need to run is the command:

bower install potion

After Bower has done it's magic, you will need to move everything from the root of the bower_components/potion folder to the main root of your folder.

Your folder structure should now look like this:

  • > bower_components/
  • > bower.json
  • > components/
  • > css/
  • > favicon.ico
  • > gulpfile.js
  • > humans.txt
  • > img/
  • > index.html
  • > js/
  • > package.json
  • > partials/
  • > README.md
  • > robots.txt

Now you can run npm install and bower install and you are ready to go!

If you do not have some of these dependencies installed like node, gulp, etc.. you can find more detailed instructions below.

What's included

Potion's core includes a standardized set of modules: base, global and modules.

Base:

  • grids.scss: Core code for creating the responsive grid system behind Potion
  • helpers.scss: A set of OOCSS (Object oriented CSS) helper classes to use throughout your HTML templates for quick margins, font styles, and more
  • main.scss: A catch all style sheet for global styles
  • normalize.scss: A set of styles used for any project to normalize your baseline CSS and HTML
  • reset.scss: Global resets to remove standard browser padding, margins, etc
  • type.scss: Typography styles

Global:

  • mixins.scss: Custom mixins for Potion used by all other .scss files
  • vars.scss: Potion's variables used by all other .scss files. These can be overridden to change global colors, fonts, sizes, etc.

Modules:

  • accordions.scss: Classes for styling Potion's custom accordions
  • alerts.scss: Classes for styling basic alerts
  • buttons.scss: Classes for Potion's custom button classes
  • code.scss: Classes for styling code blocks within your HTML
  • forms.scss: Classes for styling forms and form elements
  • headings.scss: Classes for styling h1,h2,h3,h4,h5,h6 tags
  • icons.scss: Classes for Potion's built in custom icon set
  • images.scss: Classes for images and image positioning
  • lists.scss: Classes for styling ordered and unordered lists
  • magictabs.scss: Classes for Potion's responsive tab system
  • modals.scss: Classes for styling Potion's custom modals and pop-ups
  • nav.scss: Classes for styling desktop and mobile navigation
  • panels.scss: Classes for styling alert panels
  • rules.scss: Classes for styling hr tags and dividers
  • superheros.scss: Classes for styling hero elements and banners
  • tables.scss: Classes for styling Potion's responsive table system
  • tooltips.scss: Classes for styling tooltips

Installing Node

Potion uses Node to manage your app dependencies for utilities like Gulp, etc. To install Node, visit nodejs.org/download/ and download the installer package. After you have completed the install, you can now use commands the npm install command:

npm install yourmodulehere

You will need to use the npm install command below to install things like Gulp and Bower.

Installing Gulp

Gulp is the task runner that Potion leverages for build process and efficiency. It helps streamline the build process, compresses and minifies your code and processes SASS, etc. To install Gulp, you need to have first downloaded and installed node.js (which includes npm).

Then, from the command line you can install the gulp-cli globally with:

npm install -g gulp

Change directory (cd/) to the root directory of your project and then use the command:

npm install

This command will use the package.json file within Potion to automagically install the necessary local dependencies for the project. When that process is completed, you'll be able to run the various Gulp commands provides from the command line. The default command is:

gulp

This is the command which runs all gulp tasks, creates a watch process and recompiles everything on every change to your code. You can use Ctrl C to cancel the task at any time.

Installing Bower

Bower is a front end dependency manager to take care of all your libraries like jQuery, Angular, etc. To install Bower, visit bower.io or you can install globally via command line with:

npm install -g bower

After bower is installed globally, switch to the root of your new App fodler and from the command line type:

bower install potion

You can now use the instructions from the Quick Install at the top of the page to finish up the process.

That's it! You're now ready to use Potion.

You can now run gulp from the command line and start building the next Facebook!

A note about browser support.

Potion endeavor's to support as many browsers and browser versions as possible. Most modern browsers are fully supported, however due to Internet Explorer's limited support of flexbox, custom elements, etc., Potion only supports IE9 and above. If your project or app is required to support IE8 or below, then Potion probably isn't for you :(
